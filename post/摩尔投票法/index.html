摩尔投票法

· 2021-11-26 13:09:02 · # 算法 # LeetCode

摩尔投票法也被称为极限一换一 ,

主要用来求某个容器中超过总数一半以上的元素

剑指 Offer 39. 数组中出现次数超过一半的数字

数组中有一个数字出现的次数超过数组长度的一半，请找出这个数字。 

你可以假设数组是非空的，并且给定的数组总是存在多数元素。 

示例 1:

输入: [1, 2, 3, 2, 2, 2, 5, 4, 2]
输出: 2

其算法思想主要在于 , 让数组中的两数进行比较 , 若两数不同 , 则同归于尽 , 到最后剩下的数便是数组中超过半数的数字

为此我们可以搭一个擂台 , 让数字进行比较 , 并且记录擂主剩余兵力

int table = 0; //擂台
int count = 0; //擂主剩余兵力

  • 擂台没人 , 1 为擂主 , 兵力为 1

  • 2 去挑战 , 与 1 同归于尽 , 台上仍为 1 , 但 兵力为 0

  • 3 去挑战 , 1 无兵力 , 3 直接为擂主

  • 2 去挑战 3 , 同归于尽 , 3 为擂主 , 兵力为 0

    ......

​ 最后仍在台上的即为超过半数的数字 (此时 count > 0) , 若 count == 0 , 则该数的数量正好为数组的一半

最后附上 Java 完整代码

class Solution {
    public int majorityElement(int[] nums) {
        int count = 0;
        int table = 0;
        for (int i = 0; i < nums.length; i++) {
            if (count == 0) {
                table = nums[i];
                count++;
                continue;
            }
            if (table != nums[i]) {
                count--;
            } else {
                count++;
            }
        }
        return table;
    }
}
